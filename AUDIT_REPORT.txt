PART 1: IMAGE AUDIT
=============================================================================

FINDINGS SUMMARY:
-----------------

Total HTML files analyzed: 54 (excluding templates)
Total images found: ~260+ across all pages

KEY ISSUES IDENTIFIED:

1. MISSING WIDTH/HEIGHT ATTRIBUTES
   ----------------------------------
   Severity: MEDIUM
   Impact: Can cause Cumulative Layout Shift (CLS), affecting page performance and user experience

   Most affected files:
   - blog/index.html: 12 images missing dimensions
   - services.html: 8 images missing dimensions
   - services/* pages: 4 images each missing dimensions

   Affected service pages:
   - services/transport.html
   - services/supported-independent-living.html
   - services/ndis-cleaning.html
   - services/meal-preparation.html
   - services/drop-in-support.html
   - services/daily-living.html
   - services/community-access.html
   - services/capacity-building.html

   Example issue:
   - File: services/daily-living.html, Line 1737
     <img src="../sil-dinner.webp" alt="Daily Living Support..." loading="eager">
     MISSING: width and height attributes

2. NOT USING WEBP FORMAT (OPTIMIZATION ISSUE)
   -------------------------------------------
   Severity: LOW
   Impact: PNG/JPG files are larger than webp, slowing page load

   Files with PNG/JPG instead of webp:
   - index.html: 4 PNG files (logo)
   - blog/index.html: 7 PNG files (mostly asina-logo.png)
   - supported-employment.html: 5 PNG files
   - vacancies.html: 3 PNG files
   - testimonials.html: 3 PNG files
   - terms.html: 3 PNG files
   - services/transport.html: 3 PNG files
   - And others...

   NOTE: These are primarily the "asina-logo.png" file, which is reused
   across pages. Converting this single PNG to webp would fix many issues.

   One non-logo PNG found:
   - blog/index.html uses "gardenbed.jpg"

3. MISSING LAZY LOADING
   ----------------------
   Severity: LOW
   Impact: Loads all images on page load, even those below fold

   Affected files (missing loading="lazy" on above-fold images):
   - testimonials.html: 3 images
   - index.html: 3 images
   - vacancies.html: 2 images
   - terms.html: 2 images
   - supported-employment.html: 2 images
   - Various service pages: 2 images each

   NOTE: Most below-fold images already have loading="lazy", so this is
   a minor issue affecting primarily hero/header images.

4. ALT TEXT QUALITY
   -----------------
   Severity: NONE FOUND
   Impact: Accessibility
   Status: EXCELLENT - All images have descriptive alt text

   Examples of good alt text:
   - "Community access bowling outing with Asina Disability Support worker..."
   - "Person-centred disability care worker assisting participant..."
   - "Ben Feenan, Founder of Asina Disability Support"

   No missing alt attributes or generic alt text (like "image" or "photo")
   found across the entire site.

PRIORITY RECOMMENDATIONS (by impact):

1. [MEDIUM PRIORITY] Add width/height attributes to images

   Benefit: Eliminates Cumulative Layout Shift (CLS)

   Example fix:
   FROM: <img src="../sil-dinner.webp" alt="..." loading="eager">
   TO:   <img src="../sil-dinner.webp" alt="..." width="1200" height="600" loading="eager">

   Benefit: Reduces file size, improves page speed
   Impact: Affects 15+ pages but is a single file conversion

3. [LOW PRIORITY] Add loading="lazy" to above-fold images (where appropriate)
   Benefit: Marginal performance improvement
   Impact: Affects ~10-15 images
